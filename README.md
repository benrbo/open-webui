פיצ׳ר 1:
זיהיתי את מיקום השדה של ה-System Prompt – הוא נמצא בתוך האובייקט params.
העברתי את אובייקט params מהקומפוננטה Chat אל הקומפוננטה MessageInput.
בתוך MessageInput יצרתי textarea עם bind:value={params.system}, כך שניתן לשנות את הערך באופן ישיר.
השינוי בשדה מתעדכן גם ב-Chat Controls, כך שהערך שנשלח לשרת ב-completions משקף את העריכה של המשתמש.
המשתמש יכול כעת להוסיף או לערוך System Prompt באופן נוח ומובן, מבלי להזדקק ל־chat controls של המנהלים.
הערך של System Prompt נשלח למודל כחלק מהבקשה.

פיצ׳ר 2:
הוספתי מטמון גלובלי knowledgeCache באמצעות writable מ־Svelte:

import { writable } from 'svelte/store';

export const knowledgeCache = writable<Record<string, any>>({});

כל שינוי במאגרים (הוספה/מחיקה/עדכון) מעדכן את המטמון, כך שהמסך תמיד משתמש בנתונים המעודכנים, ללא צורך בקריאות חוזרות לשרת.
טעינת תוכן הקבצים משתמשת ב־fileContentCache פנימי, שמונע קריאות חוזרות לשרת לאותו קובץ.
כל החיפוש מתבצע על המטמון בלבד (באמצעות Fuse על knowledge.files מה־cache), כך שאין צורך לשלוח בקשות חדשות בעת הקלדה או חיפוש.
שאר הפיצ’רים במסך נשמרו כמו שהם – כולל DropZone, Upload, Fuse לחיפוש ו-Modals.
המסך נטען ומגיב במהירות, גם עם מאגר גדול של קבצים.
אין בקשות מיותרות לבקאנד, וכל שינוי במידע מתעדכן באופן מיידי בממשק.
